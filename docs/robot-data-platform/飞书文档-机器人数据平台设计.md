一、文档说明

1.1 版本与范围

版本：v1.2
范围：原始数据（raw）与资产数据（asset）的路径管理、ES 检索与标签、看板展示
定位：设计文档与代码实现分离

1.2 核心分工（必记）

| 存储 | 职责 | 不做什么 |
| TiDB | 登记数据路径、流水线状态、路径血缘 | 不存标签、不存检索元数据 |
| Elasticsearch | 检索、聚合、标签（tag_paths） | 不做事务状态权威 |
| 文件系统 | 物理数据与 episode_meta.json 等 | — |
| 标签树 YAML 配置 | UI 展示与 path 校验 | 不进任何数据库 |

1.3 设计目标

| 目标 | 方案 |
| 管理库极简 | TiDB 仅 4 张表，核心是 storage_path |
| 标签灵活检索 | 标签只写 ES；大类之间 OR；同大类内多选用户可选 AND / OR |
| 检索性能 | 列表 / 筛选 / 看板走 ES |
| 路径可追溯 | TiDB 路径登记 + asset_raw_link |
| 与 manifest 对齐 | manifest_ref / storage_path 对应 manifest path |

1.4 数据流一句话

检测入库 → TiDB 登记路径 + 状态
预处理打标 → 写 tags 到 ES（或 sidecar tags.json，Worker 索引进 ES）
列表检索 → 只查 ES（含 tag_paths）
详情 → TiDB 查路径 + 磁盘读 meta + ES 补标签与统计

二、架构设计

2.1 逻辑架构（ASCII）

【文件系统】
  /data/raw/.../episode_007
  episode_meta.json
  tags.json（可选）
  /data/lerobot/...

        ↓ 检测 / 清洗 / 预处理

【TiDB 路径管理库】
  raw_data ──► es_sync_outbox
  asset_data ──► es_sync_outbox
  asset_raw_link（血缘）

        ↓ Worker 读路径 + 磁盘

【Elasticsearch】
  raw_data_records（含 tag_paths）
  asset_data_records（含 tag_paths）

        ↓

【前端】
  标签树 ← tag-tree.yaml（配置，非 DB）
  列表 / 看板 ← ES
  路径状态 ← TiDB

2.2 存储职责

| 组件 | 存什么 | 典型操作 |
| TiDB | storage_path、manifest_ref、metadata_uri、status | 登记路径、推进状态、血缘 |
| ES | 可检索字段 + tag_paths + 从 meta 冗余的统计 | search、aggs、更新标签 |
| tag-tree.yaml | 树结构、显示名、category | 前端渲染、path 校验 |
| 磁盘 meta | episode_meta、info.json、tags.json | Worker 索引时读取 |

2.3 读写路径

【写入】
  检测 → INSERT raw_data(storage_path, manifest_ref, status=init)
  预处理完成 → UPDATE status=finished → Worker 读 metadata_uri + tags.json → index ES
  人工改标签 → 直接 UPDATE ES 文档 tag_paths（TiDB 不变）

【读取】
  列表 / 筛选 / 看板 → ES
  路径是否存在 / 当前状态 → TiDB
  单条完整详情 → TiDB 路径 + 读磁盘 meta + ES 文档合并

三、业务流程与状态机

3.1 原始数据状态机

init（检测入库，登记路径）
  → correct（清洗通过）/ anomaly（清洗失败）
  → processing（预处理开始）
  → finished（预处理完成，索引 ES）
  → archived（冷归档，可选）

| 状态 | 触发 | TiDB | ES |
| init | 扫描到新路径 | INSERT path | — |
| correct / anomaly | 清洗 | UPDATE status | — |
| processing | 打标开始 | UPDATE status | — |
| finished | 打标完成 | UPDATE status | index/update，写入 tag_paths |
| archived | 冷存储 | UPDATE status | 可选 delete 或保留 |

3.2 资产数据状态机

init（合成任务登记资产路径）
  → processing（开始 build）
  → success（合成成功）/ failure（合成失败）
  → published（发布训练）

| 状态 | TiDB | ES |
| init | INSERT asset path | — |
| success / published | UPDATE status | index，写入 tag_paths + source_raw_data_ids |

3.3 标签写入时机（仅 ES，TiDB 不涉及）

| 阶段 | 标签写入方式 |
| 自动打标（预处理） | 输出 tags.json 或写 ES；Worker 索引时带入 tag_paths |
| 人工修正 | API 直接更新 ES 文档 tag_paths |
| 资产打标 | 合成完成后同上，仅 ES |

四、TiDB 表结构设计（路径管理库）

完整 DDL 见仓库：docs/robot-data-platform/appendix-ddl.sql

4.1 表清单（仅 4 张表）

| 表名 | 说明 | 核心字段 |
| raw_data | 原始数据路径登记 | storage_path UNIQUE |
| asset_data | 资产路径登记 | storage_path UNIQUE |
| asset_raw_link | 资产 ← 原始路径血缘 | raw_storage_path |

| 字段 | 类型 | 约束 | 说明 |

| uuid | CHAR(36) | UNIQUE NOT NULL | 对外 ID，ES _id 同源 |
| storage_path | VARCHAR(1024) | UNIQUE NOT NULL | 物理根路径，管理主键 |
| manifest_ref | VARCHAR(512) | INDEX | manifest.jsonl 的 path |
| metadata_uri | VARCHAR(1024) | | episode_meta.json 绝对路径 |
| status | VARCHAR(32) | NOT NULL | init/correct/anomaly/processing/finished/archived |
| data_type | VARCHAR(32) | NOT NULL | episode_dir / ros_bag / mp4 等 |
| source_type | VARCHAR(32) | NOT NULL | collected / open_source 等 |
| es_sync_version | BIGINT | | 变更版本 |
| es_indexed_at | DATETIME | | 最近索引时间 |
| created_at / updated_at | DATETIME | | |

刻意不存（改由 ES / 磁盘 meta 承担）：name、task_name、robot_id、total_frames、tag_*、描述文本等。

4.3 asset_data 资产路径表

| 字段 | 类型 | 约束 | 说明 |
| id | BIGINT | PK | |
| uuid | CHAR(36) | UNIQUE NOT NULL | ES _id |
| storage_path | VARCHAR(1024) | UNIQUE NOT NULL | 资产根路径 |
| dataset_id | VARCHAR(256) | INDEX | 逻辑 ID，如 local/pick-place-v1 |
| metadata_uri | VARCHAR(1024) | | meta/info.json |
| status | VARCHAR(32) | NOT NULL | init/processing/success/failure/published |
| asset_type | VARCHAR(32) | NOT NULL | lerobot_dataset 等 |
| es_sync_version / es_indexed_at | | | 同步控制 |
| created_at / updated_at | DATETIME | | |

4.4 asset_raw_link 路径血缘

| 字段 | 类型 | 说明 |
| asset_id | BIGINT FK | 资产 |
| raw_id | BIGINT FK | 原始数据 |
| raw_storage_path | VARCHAR(1024) | 冗余原始路径，支持按路径反查资产 |

约束：UNIQUE(asset_id, raw_id)

4.5 es_sync_outbox

| 字段 | 类型 | 说明 |
| entity_type | VARCHAR(32) | raw_data / asset_data |
| entity_id | BIGINT | |
| entity_uuid | CHAR(36) | |
| storage_path | VARCHAR(1024) | Worker 定位磁盘 |
| op | VARCHAR(16) | index / update / delete |
| payload | JSON | 可选预组装文档；空则 Worker 从路径读取 |
| status | VARCHAR(16) | pending / done / failed |

4.6 TiDB 使用注意

| 项 | 建议 |
| 引擎 | TiDB 7.x+，MySQL 协议 |
| 主键 | AUTO_INCREMENT；高并发可用 AUTO_RANDOM |
| 事务 | 路径登记 + outbox 同一事务 |
| 字符集 | utf8mb4 |
| 路径列 | 长度 1024；建 UNIQUE 前统一规范化（无尾斜杠） |

五、标签设计（仅存 ES）

5.1 原则

| 项 | 说明 |
| 存储位置 | 仅 ES 文档字段 tag_paths |
| 树结构 | tag-tree.yaml 配置文件，不进 TiDB |
| 绑定关系 | 不建绑定表；tag_paths 数组即绑定结果 |
| 可选 sidecar | 预处理可写 {storage_path}/tags.json 供 Worker 读取 |

5.2 路径规范

格式：{category}/{...}/{leaf}，无 leading /
category = 首段，代表大类
示例：scene/indoor/kitchen、task/pick_red_block

5.3 查询语义（重点）

| 层级 | 逻辑 | 是否可配置 |
| 不同 category 之间 | OR | 固定（满足任一大类即可） |
| 同一 category 内多选 | AND 或 OR | 用户可选，默认 and |

组合公式：

命中 ⟺ 满足 category_A 组条件 OR category_B 组条件 OR …
每组条件 = 按该组 op 对组内 path 做 AND 或 OR

| 场景 | scene 组 | task 组 | 语义 |
| 默认 | op:and 厨房+桌A | op:and 抓红块 | (厨房且桌A) 或 抓红块 |
| 场景任选 | op:or 厨房|室外 | — | 厨房 或 室外 |
| 混合 | op:or 厨房|桌A | op:and 抓红且高质量 | (厨房或桌A) 或 (抓红且高质量) |

5.4 标签树配置示例（YAML，不进库）

raw:
  - category: scene
    name: 场景
    children:
      - path: scene/indoor/kitchen
        name: 厨房
        leaf: true

API 建议：GET /api/v1/tag-tree?domain=raw 返回 YAML 解析结果，不查数据库。

5.5 预处理输出 tags.json（可选）

{
  "tag_paths": ["scene/indoor/kitchen", "task/pick_red_block"],
  "source": "auto",
  "updated_at": "2025-07-16T22:30:00Z"
}

路径：{storage_path}/tags.json。Worker 索引 ES 时合并；不写入 TiDB。

5.6 人工改标流程

用户在前端勾选标签
  → PATCH /api/v1/raw-data/{uuid}/tags
  → 校验 path 在 tag-tree.yaml 内且为 leaf
  → 更新 ES tag_paths only
  → TiDB 无变更

六、Elasticsearch 索引设计

Mapping 见仓库：docs/robot-data-platform/appendix-es-mapping/

6.1 索引划分

| 索引名 | 写入时机 | _id |
| raw_data_records | raw status=finished 或标签/meta 变更 | uuid |
| asset_data_records | asset success/published 或标签变更 | uuid |

通用约定：dynamic strict；refresh_interval 5s；_id = uuid 幂等 upsert

6.2 字段分层

| 层级 | 字段来源 | 示例 |
| 关联 TiDB | Worker 从 TiDB/outbox 带入 | uuid, storage_path, manifest_ref, status |
| 磁盘 meta | 读 metadata_uri | name, task_name, robot_id, total_frames, collected_at |
| 标签 | tags.json 或打标程序 | tag_paths |
| 血缘 | asset_raw_link + TiDB | source_raw_data_ids, source_raw_data_uuids |

6.5 主要 ES 字段（raw_data_records）

| 字段 | ES 类型 | 用途 |
| uuid | keyword | 文档 ID |

| manifest_ref | keyword | 过滤 |
| status / data_type / source_type | keyword | 过滤、聚合 |

| collected_at 等 | date | 范围、趋势 |
| sync_version | long | 幂等校验 |

6.6 asset 额外字段

dataset_id、episode_count、frame_count、source_raw_data_ids、source_raw_data_uuids、audited_at、synthesized_at、published_at

七、检索设计

7.1 场景矩阵

| 场景 | 数据源 |
| 多维列表筛选（含标签） | ES |
| 看板聚合 | ES |
| 路径是否已登记 / 当前状态 | TiDB |
| 标签树 UI | tag-tree.yaml |
| 改标签 | ES only |
| 资产血缘 | ES source_raw_data_ids 或 TiDB asset_raw_link |

7.2 标签组合查询（核心）

请求结构：每个 category 带 paths + op（and | or，默认 and）

{
  "tag_filter": {
    "scene": {
      "paths": ["scene/indoor/kitchen", "scene/indoor/table_a"],
      "op": "and"
    },
    "task": {
      "paths": ["task/pick_red_block", "task/place_blue_block"],
      "op": "or"
    }
  }
}

语义：(厨房 AND 桌A) OR (抓红块 OR 放蓝块)

7.2.1 对应 ES DSL

{
  "query": {
    "bool": {
      "should": [
        {
          "bool": {
            "must": [
              { "term": { "tag_paths": "scene/indoor/kitchen" } },
              { "term": { "tag_paths": "scene/indoor/table_a" } }
            ]
          }
        },
        {
          "bool": {
            "should": [
              { "term": { "tag_paths": "task/pick_red_block" } },
              { "term": { "tag_paths": "task/place_blue_block" } }
            ],
            "minimum_should_match": 1
          }
        }
      ],
      "minimum_should_match": 1
    }
  }
}

7.2.2 单大类、组内 OR

请求：
{
  "tag_filter": {
    "scene": {
      "paths": ["scene/indoor/kitchen", "scene/outdoor"],
      "op": "or"
    }
  }
}

DSL：
{
  "query": {
    "bool": {
      "should": [
        { "term": { "tag_paths": "scene/indoor/kitchen" } },
        { "term": { "tag_paths": "scene/outdoor" } }
      ],
      "minimum_should_match": 1
    }
  }
}

7.2.3 单大类、组内 AND

{
  "query": {
    "bool": {
      "must": [
        { "term": { "tag_paths": "scene/indoor/kitchen" } },
        { "term": { "tag_paths": "scene/indoor/table_a" } }
      ]
    }
  }
}

7.2.4 DSL 构建规则

| 步骤 | 规则 |
| 1 | 每个 category → 一个子句 buildCategoryClause(paths, op) |
| 2 | op=and → bool.must；op=or → bool.should + minimum_should_match:1 |
| 3 | 仅 1 个 path → 直接 term，忽略 op |
| 4 | 多个 category → 外层 bool.should + minimum_should_match:1 |
| 5 | 仅 1 个 category → 直接返回该组子句，不加外层 should |

7.3 子树检索

点击树节点 scene/indoor：

{ "prefix": { "tag_paths": "scene/indoor/" } }

尾斜杠避免 scene/indoor 误匹配 scene/indoor2。

7.4 按路径精确查

ES：{ "term": { "storage_path": "/data/raw/.../episode_007" } }
TiDB：SELECT * FROM raw_data WHERE storage_path = ?

7.5 看板聚合示例

组内逻辑：每个 category 标题旁下拉「且(AND)」/「或(OR)」；勾选 ≥2 个叶子时生效。

9.3 详情页分区

| 区块 | 数据来源 |
| 路径信息 | TiDB：storage_path、manifest_ref、metadata_uri、status |
| 采集元数据 | 磁盘 episode_meta.json |
| 标签 | ES tag_paths；编辑只调 ES API |
| 关联资产 | ES 或 TiDB asset_raw_link |

9.4 标签筛选交互

| 操作 | 行为 |
| 勾选叶子 | 加入该 category 组 |
| 切换且/或 | 更新该组 op，立即重查 ES |
| 组内 AND | 数据须同时含组内所有已选 tag |
| 组内 OR | 数据含组内任一已选 tag 即可 |
| 跨组 | 固定 OR |
| 详情页保存标签 | PATCH ES，不写 TiDB |

9.5 总览 KPI

| 卡片 | 来源 |
| 已登记路径数 | TiDB COUNT(*) |
| 可检索 finished 数 | ES status=finished |
| 标签分布 TopN | ES aggs tag_paths |
| 磁盘占用 | 扫描脚本 |

9.6 列表检索时序

浏览器 → API：search + tag_filter

ES → API → 浏览器：列表

浏览器 → API：GET detail uuid

API → 浏览器：合并详情

十、同步与一致性

10.1 Outbox 流程

1. BEGIN（TiDB 事务）
2. UPDATE raw_data SET status='finished', es_sync_version=es_sync_version+1
3. INSERT es_sync_outbox(storage_path, entity_uuid, op='index')
4. COMMIT
5. Worker：读 storage_path → metadata_uri + tags.json → 组装 ES 文档 → bulk index
6. UPDATE es_indexed_at, outbox.status='done'

10.2 标签变更（不经 TiDB）

预处理 / 人工 → 更新 tags.json 或直接写 ES
示例：POST raw_data_records/_update/{uuid}  body: { "doc": { "tag_paths": [...] } }
TiDB 无需事务

10.3 一致性

| 场景 | 级别 |
| 路径已登记 | TiDB 强一致 |
| 列表含标签 | ES 最终一致（秒级） |
| 标签刚改完 | 直接写 ES 时读后即新 |
